Part 2

The Architecture of Trust

Top Level

The Forest

The highest-level construct. Maximum trust boundary. If an attacker owns the forest root, they own the infrastructure.

The Tree

A collection of domains sharing a namespace. Parent-child relationships create automatic, transitive trusts.

The Domain

Primary operational unit. Contains users, computers, and Domain Controllers. A domain is only as secure as its trusts.

Organizational Units (OUs)

Containers for organizing objects and applying Group Policy (GPOs). Not a security boundary!

Part 3

Organization vs. Access

A Critical Distinction

Organizational Unit

Policy Application

  • •Applies Group Policy (GPOs) for settings (scripts, config).
  • •Object can reside in only ONE OU.
  • ⚠Risk: Misconfigured OUs allow malicious GPOs.

Security Group

Access Control

  • •Manages permissions for resources (files, apps).
  • •Users can belong to MULTIPLE groups.
  • ⚠Risk: Nested groups hide privilege escalation paths.
"OUs are administrative containers focused on policy... while security groups are permission containers focused on access."
Part 4

Core Services Ecosystem

Active Directory is more than just logins. It's a suite of four distinct modules serving the enterprise.

AD Domain Services (AD DS) The Core

Handles logins, permissions, and stores objects. Primary Pentest Target.

AD Certificate Services (AD CS) PKI

Manages digital certificates (VPN, Wi-Fi). Misconfiguration leads to persistence rivaling domain compromise.

AD Lightweight Directory (AD LDS) Application

Directory capabilities for apps without domain dependencies.

AD Rights Management (AD RMS) Protection

Enforces info protection (e.g., "Do Not Print") even after files leave the network.
